# ADV-LEAN Research — Primitives-Only Core

Assumptions:

- "Core" means the package surface a consumer expects from a domain-neutral UI framework: primitives, layout primitives, tokenized styling, and accessibility interactions.
- A component's current use inside GODX apps is not evidence of core membership.
- `@godxjp/ui-app` can depend on `@godxjp/ui`, TanStack Query, React Router, app i18n, and product conventions; core should not.
- Peer citation shorthand: Radix Primitives = low-level accessible primitives, unstyled/customizable (`https://www.radix-ui.com/primitives/docs`); Base UI = foundational unstyled/headless components (`https://base-ui.com/`, `https://v5.mui.com/base-ui/all-components/`); React Aria = style-free accessible components, composition and custom patterns (`https://react-aria.adobe.com/`, lines 113-115 and 475-482 in fetched page).

Bottom line: LEAN keeps the foundation strong by separating primitive behaviors from app-runtime adapters. The core should retain Button/Input/Select/Dialog/Popover/Tooltip/Calendar/DatePicker/Card/Badge primitives and a real layout set, while `@godxjp/ui-app` owns GODX preferences, app chrome, status dictionaries, query helpers, router links, and shaped loading/dashboard recipes.
