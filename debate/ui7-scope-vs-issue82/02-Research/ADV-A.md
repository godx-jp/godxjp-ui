# ADV-A evidence dossier — Option A conservative cleanup

## Position

- Ship 7.0.0 as a tight hard-rename cleanup: remove/merge the locked legacy/domain components, accept only the canonical renames already locked or effectively unavoidable, keep `Sheet`, and defer #82 primitives to 7.x minors.
- Assumption: 7.0.0 has no deprecated aliases, so every extra rename creates immediate consumer breakage.
- Assumption: this repo is already consumed by a production app, so migration cost is not theoretical.
- Assumption: adding new primitives does not require a major; hard removals/renames do.

## Design-system facts

- Radix, shadcn/ui, Chakra, Mantine, MUI, Carbon, and Fluent all use canonical low-level names like `Progress`, `Avatar`, `Separator`, `Toggle`, `Accordion`, and `ContextMenu`; this supports #82 as a roadmap, not as a major-release requirement.
- Ant Design uses `Descriptions` for key/value detail records; this supports `KeyValueGrid` -> `Descriptions`.
- shadcn/ui and Radix use `Sheet` for side-panel/dialog-style sheet wrappers, while vaul-style `Drawer` is commonly a bottom drawer/bottom sheet; this supports keeping `Sheet` and reserving `Drawer`.
- MUI uses `AppBar`, Chakra uses `SimpleGrid`, Mantine uses `AppShell`; those facts make the audit names recognizable, but do not prove this repo's existing product-shaped components can be safely renamed without redesign.

## Confidence

- 86/100. The strongest evidence is the source shape and export surface. The only uncertainty is consumer telemetry beyond the prompt's stated production usages.
