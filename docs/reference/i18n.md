# i18n reference

`@godxjp/ui` ships a pre-configured i18next singleton with four mandatory locales.

Import from `@godxjp/ui/i18n`.

---

## `initI18n()` — signature

```ts
function initI18n(): typeof i18next
```

Call once before `createRoot(…).render(…)`. Returns the configured i18next instance.

**Behavior:**

1. Uses `i18next-browser-languagedetector` with detection order: `["localStorage", "navigator"]`.
2. Storage key: `GODX_LOCALE_STORAGE_KEY` (`"godx.locale"`).
3. Falls back to `"ja"` when no locale is detected or the detected locale is not in `SUPPORTED_LOCALES`.
4. Sets `load: "languageOnly"` — drops subtags (`en-US` → `en`).
5. Sets `interpolation.escapeValue: false` — React handles XSS escaping.

**Do not call more than once.** The i18next `init` method is idempotent but calling `initI18n()` from multiple modules is a code smell — call it once in `main.tsx`.

## Constants

| Export | Value | Description |
|---|---|---|
| `SUPPORTED_LOCALES` | `["ja", "en", "vi", "fil"]` | Four mandatory locales |
| `GODX_LOCALE_STORAGE_KEY` | `"godx.locale"` | localStorage key for locale preference |
| `FORGE_LOCALE_STORAGE_KEY` | `"godx.locale"` | Deprecated alias — same value as above |

## `addResourceBundle` — service namespace extension

```ts
import i18n from "@godxjp/ui/i18n"

// After initI18n() is called:
i18n.addResourceBundle("ja",  "my-svc", { title: "マイサービス" })
i18n.addResourceBundle("en",  "my-svc", { title: "My Service" })
i18n.addResourceBundle("vi",  "my-svc", { title: "Dịch vụ" })
i18n.addResourceBundle("fil", "my-svc", { title: "Aking Serbisyo" })
```

Use the namespace in components:

```tsx
import { useTranslation } from "react-i18next"

const { t } = useTranslation("my-svc")
t("title")  // → "My Service" (or locale equivalent)
```

## Fallback rules

1. `localStorage["godx.locale"]` (user override).
2. `navigator.language` prefix (e.g. `en-US` → `en`).
3. `"ja"` if nothing matches.

Missing keys fall back to `"ja"` (i18next fallbackLng). Missing keys in `"ja"` render the raw key string.
