# Explanation

Explanation docs are understanding-oriented. They provide background, rationale,
and analysis — the "why" behind decisions. They do not instruct you to do anything;
read them to deepen your mental model.

| Document | What it explains |
|---|---|
| [Overview](./overview.md) | What `@godxjp/ui` is, who it is for, what it is not |
| [Design philosophy](./design-philosophy.md) | Three founding pillars: Radix, shadcn-style, tokens-not-utilities |
| [Architecture](./architecture.md) | Atom → molecule → organism layer model with diagram |
| [Accessibility](./accessibility.md) | WCAG 2.1 AA baseline and cross-cutting patterns |
| [Preferences](./godx-config.md) | The `GodxConfigProvider` — locale + timezone propagation |

For the GoDX visual language (shibumi / ma / kanso, wa-iro palette, OKLCH chroma
cap) see [`BRAND.md`](../../BRAND.md). For binding architectural specs (token
system, theme axes, prop vocabulary, consumer contract, design handoff formats)
see [`docs/specs/`](../specs/README.md).
