勤怠ドメインのルール: ① 日付フォーマットは YYYY/MM/DD · 曜日 固定 — ISO ハイフン (2026-05-09) は API/コード用、UI では使わない。 ② 時刻は 24時間表記固定 — AM/PM はシフトの境目で誤読を生む (例: 22:00 vs 10PM)。 ③ 分単位は 5分丸めを default — 打刻の生秒は 背景情報として小さく併記。 ④ 期間ピッカーは必ず 日数 / 平日数 / 祝日数を footer に出す — 休暇申請の文脈で「結局何日休む?」が即読める。 ⑤ Disabled な日は 取り消し線 + ツールチップで理由 (入社日前 / 退職日後 / 別店舗で既出勤) を出す — silent で消すと「なぜ選べない?」のクレームになる。 ⑥ 夜勤 (日跨ぎ) は datetime ペアで持つ — date + time の独立 2 フィールドは 翌日 06:00 の表現で必ず壊れる。
